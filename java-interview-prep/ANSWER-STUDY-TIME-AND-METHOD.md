## 📚 Phương Pháp Học Hiệu Quất Nhất

### **Method 1: Code → Explain → Verbalize**

**Sai:**
```
1. Read theory
2. Read code example
3. Summarize mentally
4. Move on
→ Retention: 40%
```

**Đúng (5-step process):**
```
STEP 1: CODE FIRST (5 min)
        Đọc code example TRƯỚC lý thuyết
        Tự hỏi: "Cái này là cái gì?"
        
STEP 2: EXPLAIN (5 min)
        Đọc explanation/answer
        So sánh với thinking của bạn
        
STEP 3: VERBALIZE (5 min)
        Đóng file. Nói to:
        "Cái này là..., vì..., dùng khi..."
        THIS IS CRITICAL — talking forces organization
        
STEP 4: TEST NEXT DAY
        Flashcard: same topic
        Answer từ memory (không nhìn notes)
        
STEP 5: REPEAT WEEK LATER
        Spaced repetition at 1, 3, 7, 14 days
        
→ Retention: 80%
```

### **Method 2: Spaced Repetition (Lặp lại theo khoảng cách)**

**Timeline mà brain nhớ tốt nhất:**

```
Day 1:   Learn topic (read + understand)
Day 2:   Review (verbalize + Q&A) → memory 70%
Day 5:   Quick quiz (3 days sau) → memory 80%
Day 12:  Revisit (7 days sau) → memory 85%
Day 30:  Final check (18 days sau) → locked in (90%+)
```

**Tool:** Anki flashcards (auto-spacing)

**Card format:**
```
Front: "What's the difference between @Transactional REQUIRED vs REQUIRES_NEW?"
Back:  "REQUIRED: share existing TX or create new.
        REQUIRES_NEW: ALWAYS create new, suspend existing.
        Use case: Audit logging must succeed even if main TX fails."

Schedule: Review this card on Day 2, Day 5, Day 12, Day 30 (auto)
```

## 🎯 Phương Pháp Cụ Thể — Các Kỹ Thuật

### **1. Elaborative Interrogation (WHY? HOW? WHEN?)**

**Vấn đề:** Bạn học "@Transactional behavior" và quên nó

**Giải pháp:** Hỏi WHY:

```
❓ WHY does @Transactional exist?
✓ To ensure DB consistency — atomicity

❓ HOW does it work?
✓ Spring wraps method in AOP proxy.
  When you call it, proxy: BEGIN TX → your code → COMMIT/ROLLBACK

❓ WHEN does self-invocation fail?
✓ When you call this.method() internally — bypasses proxy

❓ WHAT IF I use REQUIRES_NEW?
✓ New TX always created, even if one exists
```

**Why this works:** Writing "why/how/when" forces connections. Isolated facts → forgotten. Connected concepts → remembered.


Dual coding (visual + verbal) = 2x better retention than reading alone.

### **Tool 2: Zoom/Phone Recorder**

**Why:** You can't hear yourself think. Recording reveals:
- Stutters (need clarification)
- Gaps (you said "umm" for 5 seconds)
- Pace (spoke too fast, hard to follow)

**Setup:**
```

2. Record your screen
3. Read Q from file
4. Click record
5. Answer 2 min (no notes)
6. Play back, listen
7. Refine, repeat Q again
```

## ❌ Common Mistakes to AVOID

### **Mistake 1: Studying >90 min at once**
```
Brain focus drops after 50 min anyway.
3 hours of studying = first 45 min is 80% quality,
                     next 45 min is 50% quality,
                     last 90 min is 20% quality.

Better: 6 × 45min sessions = consistent 80% quality
```

### **Mistake 2: Only reading, never speaking**
```
Reading feels like understanding.
Saying out loud reveals gaps immediately.

Example: You read "@Transactional gotcha"
         You think "oh yeah, I knew that"
         You try to explain → "umm... wait"
         
→ Only speaking reveals real understanding
```

### **Mistake 3: Studying similar topics in a row**
```
Mon: HashMap
Tue: HashSet ← interference! Brain confuses them

Better: HashMap → Threads → Streams → Collections Review
        Spacing out similar topics helps distinguish them
```

### **Mistake 4: Studying 7 days/week**
```
Brain needs 1 day/week to consolidate (no new info).
6 days study + 1 day rest = better retention than
7 days study every day.

Reason: Sleep is consolidation. Body needs break from stress.
```
